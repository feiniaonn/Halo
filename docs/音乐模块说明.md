# 音乐模块说明

## 模块定位

音乐模块负责读取 Windows 当前媒体会话，呈现当前播放信息，并补齐歌词、历史统计与迷你播放器体验。它不是独立播放器，而是一个围绕系统会话做聚合、展示和增强的桌面控制层。

## 当前能力

- 监听系统媒体会话，获取当前歌曲、歌手、专辑、封面、播放状态与进度。
- 在 QQ 音乐、网易云、酷狗、酷我、LRCLib 等多源之间拉取歌词，并支持翻译、音译与候选切换。
- 记录播放历史、日汇总和 Top10，给音乐页和迷你模式提供统计数据。
- 提供浮窗、迷你播放器和主音乐页三套 UI 入口，共享同一份当前播放快照。
- 暴露设置项，包括歌词开关、偏移、自动跟随、抓取策略与展示行为。

## 技术结构

### 前端

- `src/pages/MusicPage.tsx`：音乐主页面，负责当前歌曲展示、历史、统计与歌词面板。
- `src/pages/MiniPlayerPage.tsx`：迷你播放器页面，适合悬浮使用。
- `src/components/layout/FloatingPlayer.tsx`：全局悬浮条中的音乐状态与歌词摘要。
- `src/modules/music/hooks/useCurrentPlaying.ts`：统一订阅 `music:current-changed` 事件，并在窗口可见性变化时做轻量轮询兜底。
- `src/modules/music/hooks/useMusicLyrics.ts`：管理歌词请求、候选切换、错误态与缓存命中。
- `src/modules/music/services/musicService.ts`：封装 Tauri 命令调用，包括当前播放、控制、历史、统计和歌词接口。

### 后端

- `src-tauri/src/music.rs`：核心监听逻辑，基于 Windows GSMTC 获取媒体会话快照，并在特殊场景对 QQ 音乐做兼容判断。
- `src-tauri/src/music_lyrics/`：歌词聚合层，负责多源抓取、评分、合并、缓存与超时控制。
- `src-tauri/src/music_settings.rs`：音乐相关设置的默认值、读写和约束。
- `src-tauri/src/db.rs`：播放历史、歌词缓存与统计落库。
- `src-tauri/src/lib.rs`：命令注册与后台监听启动入口。

## 数据流

1. Rust 后台监听系统媒体会话变化。
2. 会话快照变化后，后端发出 `music:current-changed` 事件。
3. 前端通过 `useCurrentPlaying` 同步当前歌曲信息，并按需请求歌词和统计数据。
4. 歌词结果先经过候选去重、优先级排序和缓存命中，再回传给 UI。
5. 音乐页、迷你页和浮窗共享同一套当前播放状态，但按各自场景渲染不同密度的信息。

## 当前边界

- 模块依赖 Windows 媒体会话能力，其他平台目前没有等价支持。
- 歌词抓取强依赖外部服务可用性，虽然已有缓存和多源兜底，但不能保证所有歌曲都能返回同步歌词。
- 部分播放器不会稳定暴露精确播放进度，迷你模式下的歌词滚动仍可能出现轻微漂移。

## 适合继续演进的方向

- 补更稳的进度校正机制，减少歌词与真实播放的时间偏差。
- 将历史统计进一步结构化，给首页或仪表盘提供更清晰的播放画像。
- 为歌词抓取补更多失败诊断，便于快速判断是源站不可用、匹配失败还是缓存命中异常。
