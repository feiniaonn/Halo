# Windows 桌面端 TVBox 解析播放器 - 需求说明文档

## 1. 项目概述
### 1.1 目标
完善项目中的点播模块，使其能够解析 TVBox 类影视配置（JSON 格式），实现与 Android 端 TVBox 类似的功能：浏览影视资源、搜索、播放视频。重点解决 Android 环境依赖问题，使大量原本只能在 Android 上运行的爬虫 JAR 包也能在 Windows 上被调用或模拟。

### 1.2 用户场景
- 用户在电脑上观看影视，希望使用 TVBox 丰富的资源站。
- 用户手中有各种 TVBox 配置文件（单仓链接或本地 JSON），希望导入后直接播放。
- 用户需要兼容现有 TVBox 生态中的各类爬虫（JAR、JS、XBPQ 等），无需为每个源单独开发。

### 1.3 核心挑战
- TVBox 爬虫 JAR 包深度依赖 Android 运行时（如 `android.*` 包、原生库 `libstub.so`），无法在普通 JVM 中运行。
- 部分 JS 爬虫（如 drpy2）也依赖 Android 特有的 Java 对象。
- 配置格式复杂，字段繁多，需完整解析并正确调用相应爬虫。

## 2. 功能需求
### 2.1 配置管理
- **导入配置**：支持输入单仓 URL（如 `https://www.iyouhun.com/tv/dc-xs`）或本地 JSON 文件路径，自动解析并显示站点列表。
- **配置解析**：正确解析 TVBox 标准 JSON 格式，包括全局字段（`spider`、`wallpaper`、`logo`、`sites`、`parses`、`lives`、`doh`、`proxy`、`rules` 等）。
- **站点显示**：以列表或网格形式展示所有站点，支持按名称搜索、排序。

### 2.2 站点浏览
- **分类加载**：点击站点后，加载该站点的分类（如电影、电视剧、综艺等），支持筛选（`filterable` 字段）。
- **列表分页**：支持翻页加载更多内容。
- **详情页**：点击影片进入详情页，显示简介、演员、剧集列表等。

### 2.3 搜索功能
- **全局搜索**：在所有支持搜索的站点中同时搜索（需考虑并发和超时）。
- **站内搜索**：在当前站点内搜索（若 `searchable` 为 1）。

### 2.4 视频播放
- **播放器集成**：集成主流播放内核（如 VLC、MPV 或 WebView2），支持常见视频格式（mp4、m3u8、flv 等）。
- **解析接口**：支持配置中的 `parses` 解析器，将视频播放页 URL 转换为直接播放地址。
- **播放规则**：支持配置中的 `rules`，用于过滤广告片段或执行点击脚本（需模拟浏览器环境）。

### 2.5 直播功能
- **直播源加载**：解析 `lives` 中的 M3U 直播源，支持分组显示。
- **EPG 节目指南**：若配置提供 EPG 模板，可显示当前节目信息。

### 2.6 设置与优化
- **DNS over HTTPS**：支持配置中的 DoH 服务器，用于优化解析。
- **代理设置**：支持配置中的 `proxy` 域名列表，可自定义代理规则。
- **Hosts 映射**：支持配置中的 `hosts`，实现域名重定向。
- **广告过滤**：支持配置中的 `ads` 域名黑名单。

## 3. 技术架构与实现方案
### 3.1 整体架构
应用采用分层设计：
- **UI 层**：使用跨平台框架（如 Electron、Flutter 或 .NET MAUI）构建 Windows 原生界面。
- **核心引擎层**：负责解析配置、管理站点、调度爬虫。
- **爬虫适配层**：处理不同类型的爬虫，提供统一接口。
- **播放层**：集成播放器，处理视频流。

### 3.2 爬虫适配方案（重点）
针对不同类型的爬虫，采用不同策略：

| 爬虫类型 | 特征 | 适配方案 |
|---------|------|---------|
| **XBPQ / JSON 规则** | `api` 字段为 `csp_XBPQ` 或类似，无 `jar` 字段 | 直接解析配置中的规则（XPath、JsonPath），使用 HTTP 请求库获取数据。 |
| **内置 CSP** | `api` 为 `csp_Douban`、`csp_Bili` 等，无 `jar` | 在应用中用原生代码实现这些内置爬虫（可参考 TVBox 源码用 Java/Kotlin 实现，然后通过 JNI 或进程调用）。 |
| **JS 爬虫 (drpy2)** | `api` 为远程 JS 地址，或带有 `ext` 参数指向 JS | 在应用中嵌入 JavaScript 引擎（如 QuickJS、V8），并模拟必要的 Java 对象（如 `java.net.URL`、`java.util.HashMap`），可参考开源项目 `tvbox-js-env`。 |
| **JAR 爬虫 (Android-only)** | `api` 为 `csp_XXX`，且包含 `jar` 字段指向 JAR 文件 | **方案 A（远程调用）**：通过 ADB 连接一个 Android 设备或模拟器，在设备上运行 TVBox 的爬虫服务，Windows 端通过 HTTP 接口调用爬虫。需在 Android 端部署一个轻量服务（如使用 TVBox 的爬虫 API）。<br>**方案 B（环境模拟）**：在 Windows 上运行完整的 Android 运行时（如使用 Android-x86 容器或 Anbox），但复杂度高，不建议初版实现。<br>**方案 C（请求模拟）**：抓包分析 JAR 发起的网络请求，在 Windows 端直接模拟这些请求，绕过 JAR 执行。适用于加密简单的站点。 |

**初版建议**：优先支持 XBPQ、内置 CSP 和 JS 爬虫，对于需要 JAR 的站点，可暂时提示“暂不支持”，或在后期实现远程调用方案。

### 3.3 配置字段映射
应用需正确解析 TVBox 配置中的站点字段（参考用户提供的参数说明）：

| 字段名 | 说明 | 在应用中的使用 |
|-------|------|---------------|
| `key` | 站点唯一标识 | 用于内部索引和缓存。 |
| `name` | 显示名称 | 界面展示。 |
| `type` | 固定为 3（影视）或 0（其他） | 决定如何处理。 |
| `api` | 爬虫标识 | 用于匹配对应的爬虫适配器。 |
| `searchable` | 是否支持搜索 | 控制搜索功能是否启用。 |
| `quickSearch` | 是否支持快速搜索 | 同上。 |
| `filterable` | 是否支持筛选 | 分类页面是否显示筛选控件。 |
| `jar` | JAR 文件地址 | 若存在，则按 JAR 爬虫处理。 |
| `ext` | 扩展参数 | 传递给爬虫适配器的额外配置（如多域名、密钥）。 |
| `playUrl` | 播放解析地址 | 用于视频播放前的解析。 |
| `click` | 点击脚本 | 在需要时模拟点击（需内置浏览器引擎）。 |
| `categories` | 自定义分类 | 覆盖站点的默认分类。 |

### 3.4 播放处理
- 当用户点击剧集时，应用需根据 `api` 类型获取播放 URL（可能需调用爬虫的 `play` 方法）。
- 如果站点配置了 `playUrl`，则优先使用该解析接口拼接视频页 URL 获取直链。
- 如果配置中有 `rules`，则对获取到的播放列表（M3U8）应用正则过滤，移除广告片段。
- 最终将直链交给播放器内核播放。

### 3.5 网络与安全
- 所有网络请求需支持自定义 Headers（参考配置中的 `headers`）。
- 支持 DoH 和代理，提升稳定性。
- 对下载的 JAR 文件进行 MD5 校验（若配置提供了 `spider` 或 `jar` 的 MD5），防止篡改。

## 4. 用户界面示意
### 4.1 主界面
- 左侧：站点列表（可折叠分组），上方有配置导入按钮。
- 右侧：当前站点的分类/影片列表，支持网格/列表切换。
- 顶部：全局搜索框、设置入口。

### 4.2 详情界面
- 影片海报、简介、评分等信息。
- 剧集列表（按分集或线路分组）。
- 播放按钮。

### 4.3 播放界面
- 视频窗口，下方播放控制。
- 支持选集、切换线路（若站点提供）。

### 4.4 设置界面
- 导入配置（URL 或文件）。
- 代理设置、DoH 选择。
- 关于与更新。

## 5. 开发阶段建议
### 5.1 第一阶段：核心框架与基础爬虫
- 实现配置解析与存储。
- 实现 HTTP 请求模块，支持 Headers、Cookies。
- 实现 XBPQ 规则解析器（XPath/JsonPath）。
- 实现内置 CSP（如 Douban、Bili）的基础功能。
- 集成播放器（如 libmpv）。
- 完成基本 UI：站点列表、分类列表、详情页、播放。

### 5.2 第二阶段：JS 爬虫支持
- 嵌入 JavaScript 引擎，模拟 Java 基础类。
- 实现 drpy2 风格的 JS 爬虫适配。
- 支持远程 JS 脚本加载。

### 5.3 第三阶段：JAR 爬虫远程调用（可选）
- 开发 Android 端服务应用，暴露爬虫 API（可用 TVBox 源码改造）。
- Windows 端通过 ADB 或网络与设备通信，调用爬虫。
- 支持自动启动模拟器或连接已有设备。

### 5.4 第四阶段：优化与扩展
- 支持直播源。
- 支持 DoH、代理。
- 广告过滤规则实现。
- 性能优化与内存管理。

## 6. 技术选型建议
- **爬虫执行**：对于 JS 爬虫，可使用 `vm2` 沙箱；对于 Java 代码，可考虑通过子进程调用 Java 程序（如将 JAR 中的类单独提取出来运行），但需处理 Android 依赖。
- **播放器**：`mpv.js` 或 `node-mpv` 封装，或使用 VLC 的 ActiveX/COM 接口。
- **网络请求**：`axios` 或 `node-fetch`，支持自定义头。
- **配置解析**：直接 JSON parse。

## 7. 注意事项
- **性能**：大量站点同时搜索需控制并发数，避免阻塞。

## 8. 附录：TVBox 配置字段速查（用户提供）
```json
{
    "key": "站点标识",
    "name": "显示名称",
    "type": 3,
    "api": "csp_类名",
    "searchable": 1,
    "quickSearch": 1,
    "filterable": 0,
    "jar": "JAR文件地址",
    "ext": "扩展参数",
    "playUrl": "解析地址",
    "click": "点击脚本",
    "categories": ["自定义分类"]
}
```
全局字段：
- `spider`: 可能为 JAR 地址带 MD5，如 `"https://xxx.jar;md5;hash"`
- `wallpaper`: 背景图
- `logo`: 应用图标
- `sites`: 站点数组
- `parses`: 解析器数组
- `lives`: 直播源
- `doh`: DNS over HTTPS 配置
- `proxy`: 代理域名列表
- `hosts`: 域名映射
- `ads`: 广告域名
- `rules`: 播放处理规则
